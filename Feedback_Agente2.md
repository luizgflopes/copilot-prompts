🧩 VISÃO DO DIA A DIA

Como é o trabalho de um(a) Product Analytics / Data-Driven PM:

Analisar dados de uso do produto para identificar oportunidades de melhoria

Definir métricas (KPIs) e acompanhar resultados de funcionalidades

Trabalhar com times de negócio, tecnologia e dados para priorizar decisões

Criar dashboards e relatórios para dar visibilidade aos dados

Formular hipóteses e validar com experimentos (A/B tests)

Traduzir dados em decisões estratégicas de produto

🧠 MAPA DE SKILLS
CORE SKILLS (essenciais):

Análise de dados (SQL, interpretação de métricas)

Product thinking (métricas, hipóteses, priorização)

Comunicação orientada a dados (data storytelling)

NICE-TO-HAVE (complementares):

Experimentação (A/B testing)

Noções de estatística aplicada

FERRAMENTAS E TECNOLOGIAS:

SQL

Excel / Google Sheets

Power BI ou Tableau

Python (pandas – opcional, mas diferencial)

Google Analytics / ferramentas de produto

📅 ROADMAP DE 90 DIAS

ADAPTADO PARA: 6 horas/semana

MÊS 1 - FUNDAMENTOS
SEMANA 1-2:

Revisar SQL básico (SELECT, JOIN, GROUP BY)

Entender conceitos de métricas de produto (DAU, MAU, churn, conversão)

SEMANA 3-4:

Criar primeiros dashboards simples (Excel ou Power BI)

Estudar conceitos de Product Analytics (funil, cohort, retenção)

MÊS 2 - PRÁTICA
SEMANA 5-6:

Resolver problemas práticos com SQL (análise de dados reais ou simulados)

Construir análises de negócio (ex: por que usuários abandonam um fluxo?)

SEMANA 7-8:

Criar um mini projeto de análise (ex: comportamento de usuários)

Aprender fundamentos de data storytelling (como apresentar insights)

MÊS 3 - PORTFÓLIO E PREPARAÇÃO
SEMANA 9-10:

Desenvolver projeto completo de portfólio

Criar dashboard profissional + narrativa de insights

SEMANA 11-12:

Preparar apresentação do projeto (como se fosse para empresa)

Treinar entrevistas e perguntas de produto + dados

🚀 PROJETO DE PORTFÓLIO
PROJETO:

Análise de Funil de Conversão de Produto

O QUE FAZER:

Você vai simular ou usar um dataset (ex: e-commerce ou app) e analisar o funil de usuários desde entrada até conversão, identificando gargalos e propondo melhorias.

ENTREGÁVEIS:

Query SQL com análise dos dados

Dashboard (Power BI ou Excel) com métricas principais

Documento ou apresentação com insights e recomendações

CRITÉRIOS DE ACEITAÇÃO:

Identificação clara de problemas no funil

Uso correto de métricas (conversão, retenção, etc.)

Recomendações baseadas em dados (não opinião)

DICA:

Pense como PM: não basta mostrar o problema — proponha solução e impacto no negócio.

📊 MINI PLANO POWER BI (8 SEMANAS)

ADAPTADO PARA: 6 horas/semana (~1h por dia útil)

🟢 SEMANA 1 — FUNDAMENTOS E PRIMEIRO DASHBOARD

Objetivo: perder o medo da ferramenta

Instalar e explorar o Power BI (interface, menus, conceitos)

Importar dados (Excel/CSV)

Criar:

Gráfico de barras

Gráfico de linha

Cartões (KPIs simples)

Publicar primeiro dashboard básico

✅ Saída da semana: dashboard simples com 3 gráficos

🟢 SEMANA 2 — MODELAGEM BÁSICA DE DADOS

Objetivo: entender como os dados se conectam

Conceito de tabelas fato x dimensão

Criar relacionamentos entre tabelas

Limpeza de dados no Power Query:

remover colunas

alterar tipos

tratar nulos

✅ Saída da semana: dataset organizado + modelo funcional

🟡 SEMANA 3 — MEDIDAS E DAX (INÍCIO)

Objetivo: sair do “arrastar e soltar” e começar a pensar

Criar medidas com DAX:

SUM

COUNT

AVERAGE

Criar KPIs:

total de vendas

quantidade de usuários

Diferença entre coluna calculada vs medida

✅ Saída da semana: dashboard com métricas calculadas

🟡 SEMANA 4 — FILTROS E INTERATIVIDADE

Objetivo: deixar o dashboard “vivo”

Slicers (filtros)

Drill-down (explorar níveis)

Interações entre gráficos

Navegação básica

✅ Saída da semana: dashboard interativo

🟡 SEMANA 5 — VISUALIZAÇÃO PROFISSIONAL

Objetivo: sair do amador e ir para nível mercado

Boas práticas de design:

hierarquia visual

cores consistentes

evitar poluição

Escolha correta de gráficos

Layout organizado

✅ Saída da semana: dashboard visualmente profissional

🟠 SEMANA 6 — DAX INTERMEDIÁRIO

Objetivo: análises mais inteligentes

CALCULATE

FILTER

Métricas como:

crescimento (%)

comparação de períodos

Introdução a inteligência temporal (time intelligence)

✅ Saída da semana: análises comparativas (ex: mês vs mês)

🟠 SEMANA 7 — STORYTELLING COM DADOS

Objetivo: virar diferencial

Como estruturar narrativa:

contexto → problema → insight → ação

Destacar insights no dashboard

Criar “títulos inteligentes” (explicando o que está acontecendo)

✅ Saída da semana: dashboard que “explica sozinho”

🔴 SEMANA 8 — PROJETO FINAL (PORTFÓLIO)

Objetivo: consolidar tudo

Projeto sugerido:

Dashboard de Análise de Produto / Usuários

Você deve:

Criar dataset (ou usar Kaggle)

Modelar dados

Criar métricas

Construir dashboard completo

Adicionar insights

📦 ENTREGÁVEIS FINAIS

Dashboard completo (Power BI)

Prints organizados (para LinkedIn/GitHub)

Explicação do projeto:

problema

análise

conclusões

⚠️ ERROS COMUNS (evite isso)

Fazer dashboard bonito sem análise

Usar gráfico errado

Não explicar os dados

Focar só na ferramenta e esquecer o negócio

Trabalhar com múltiplas tabelas

Criar métricas com DAX

Explicar insights de forma clara

Ter material de portfólio

“Se eu fosse o PM, que decisão eu tomaria com esse dashboard?”

Isso é o que vai te diferenciar de alguém que só “mexer em ferramenta”.
